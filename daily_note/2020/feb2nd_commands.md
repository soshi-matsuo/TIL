# Linuxコマンド
## diff
引数に２つのファイルを取り、内容を比較して差異を出力する。  
`2c2`のような出力結果は、第一引数のファイルの２行目が第二引数のファイルの２行目に変更されている、という意味。  
#### オプション
-u：ファイル間の差異をユニファイド形式（Gitのように+と-で変化を表現）で表示  
-s：同一だった場合にその旨を表示  
-q：同一の場合は何も表示せず、異なっていた場合はその旨のみを表示  
-c：ファイル間の差異をコンテキスト形式（変更箇所を前後数行とともに表示）で表示  
-y：比較結果を横並びで表示
## cmp
diff同様、引数に２つのファイルを取って内容を比較し、異なっている最初の文字を出力する。  
diffと違ってファイルを先頭から１バイトずつ比較していき、異なる文字を見つけたらその場所を出力して終了する。  
通常テキストファイルにはdiffを使い、バイナリファイルをcmpで比較する。
## cat
引数に１つ以上のファイル名を取り、指定されたファイルの内容を標準出力に出力する。  
標準出力なので、`>`のリダイレクトで別のファイルに内容を上書きしたり、`>>`のリダイレクトで内容を追加したりできる。  
また、コマンドの語源（concatnate)通り、`cat file1 file2`のように複数のファイルを引数に取ることで、それらの内容を結合して出力する事ができる。  
この性質とリダイレクトを組み合わせることで、複数のファイルを簡単に１つにまとめられる。  
また、`tac`コマンドも存在し、catの逆順でファイル内容を出力する。
#### オプション
-n：行番号付きで出力  
-b：空白行を飛ばして、行番号付きで出力  
-s：連続した空白行を１行の空白行にまとめて出力  
## rename(homebrew)
`mv`と違って、複数のファイルをまとめてリネームできるコマンド。  
`rename -s beforeStr afterStr fileName`または`rename 's/before/after/' fileName`の形式で名前変更。  
## nkf
ファイルの文字コードに関する操作を行うコマンド。  
デフォルトでは、標準入力or引数に与えられたファイルの内容をJISコードで標準出力する。  
#### オプション
-g：入力の文字コードを推定して出力（UTF-8とShift-JISが混在しているとBINARYと出力される点に注意）  
-w：UTF-8に文字コード変換して内容を出力  
-s：Shift-JISに文字コード変換して内容を出力  
--overwrite：ファイル名を引数に取り、元ファイルの文字コードを変換して上書き保存
